Dashboard — what lives here

The first screen every midwife and admin sees on login. Designed for speed — the most critical information in under 5 seconds. No browsing. No searching. Just what needs attention right now.

Mothers — what lives here

The full mother record system. A midwife registers, monitors, and reviews everything about each mother from one place. Check-ins live inside a mother's record — not in a separate menu.

Mother list
Midwife
All mothers sorted by severity by default (L4 first). Filter by status: Active, Discharged, Inactive. Search by name. Each row shows name, days post-delivery, last call status, severity badge.
Severity-sorted list Status filter tabs Search bar Severity badge Days post-delivery Last call status
Mother profile
Midwife
Opens when a midwife taps a mother. Full record in one view. Tabs: Overview, Calls, Check Ins, Notes. Overview shows delivery type, risk factors, medication checklist, consent status.
Profile header Tabs: Overview / Calls / Check Ins / Notes Delivery details Medication checklist Call Now button Severity trend
New discharge (registration)
Midwife
5-step flow triggered by "New Discharge" button. Sub-30 seconds. Name, phone, delivery date, delivery type, medication checklist, consent checkboxes. On submit: welcome SMS fires immediately.
Step 1: Core data Step 2: Delivery details Step 3: Medication checklist Step 4: Consent Step 5: Confirm + submit Offline cache indicator
Log check-in (inside mother profile)
Midwife
Triggered from inside a mother's profile after a human follow-up call or physical visit. Three fields: clinical observation, medication or advice given, next action dropdown. Under 60 seconds. Auto-generates SMS to mother within 1 hour.
Clinical observation field Medication / advice field Next action dropdown Auto-SMS confirmation
Calls — what lives here

The full AI call activity stream across the entire cohort. Not buried in individual records. A midwife or admin can see everything happening across all mothers in one place.

Call log
Both
All calls across the cohort. Filterable by: date range, call type (Day 3, Week 2 etc), status (completed, missed, escalated), severity outcome. Each row links to the mother profile and the call summary.
Call list with filters Status: completed / missed / escalated Date filter Call type filter Severity outcome badge Link to mother profile
Call detail
Midwife
Opens when a call row is tapped. Shows: call date and type, duration, severity classification, AI-extracted symptom tags, summary of what was discussed, and the SMS that was sent to the mother. Call Now and Log Check In buttons at the bottom.
Call summary Severity badge Symptom tags Call duration SMS sent preview Call Now button Log Check In button
Scheduled calls
Both
Upcoming calls in the next 7 days. Shows which mothers are due for which type of call. Admin can see volume planning. Midwife uses it to prepare.

Staff & Roles — what lives here

Admin-only section. Manages who can access the portal and what they can see. A midwife never sees this menu.

Staff list
Admin
All portal users at the hospital. Name, role (midwife / admin), status (active / inactive), last login. Add new staff member button. Deactivate or reassign roles from here.
Staff list Role badge Last login Add staff button Deactivate action
Role permissions
Admin
Two roles: Midwife and Admin. Midwife: view and edit assigned patients, log check-ins, acknowledge alerts. Admin: all midwife permissions plus cohort analytics, staff management, audit exports. No custom roles in v1.
Midwife role card Admin role card Permission checklist per role
